# Work Log: Scene 4 Feedback Fixes (Round 4)

## 変更の概要
画像保存機能における「描画崩れ」「チラツキ」「無駄なスペース」を根本解決するため、実装方針を「メイン画面の加工(clone)」から「専用の隠しビューのキャプチャ」に変更しました。

## 変更ファイル一覧
- `src/components/scene/ResultScene.tsx`:
    - **Hidden Export View Pattern** を導入。
    - 画面外(`left: -9999px`)に、画像生成専用の `div` を配置。
    - 幅を **800px** に固定（ユーザー要望）。
    - 必要な列（着順・名前・着差）のみをレンダリングする専用テーブルを定義。
    - `html2canvas` はこの隠し要素を対象に実行されるよう変更。これに伴い複雑な `onclone` 処理を廃止。

## 検証
- ビルド (`npm run build`) 通過確認済み。
- これにより、ユーザーの画面解像度やブラウザのズーム設定に依存せず、常に一定のレイアウト・解像度の画像が出力されます。
