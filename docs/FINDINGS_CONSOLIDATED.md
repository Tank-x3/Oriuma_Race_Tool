# 発見事項統合リファレンス

> CODE_REVIEW_BOARD.md の全発見事項を事項ナンバリングごとに統合整理したドキュメント。
> 各事項について「発見内容」「ユーザーFB」「追加調査結果」「現在ステータス」を1箇所に集約。
>
> **ステータス凡例:**
> - **有効** — 対応が必要な事項
> - **取り下げ** — 誤検出・問題なしと判定された事項
> - **修正完了** — 既に修正が完了した事項
> - **未実装（既知）** — 機能自体が未着手であり、発見事項というよりは未実装タスク
>
> **影響度表記:** 更新がある場合は `元の影響度 → 更新後の影響度` で表記

#### #1-2-3 | `(N)` 欠落時のフォールバック処理（仕様違反）

| 項目 | 内容 |
|------|------|
| **種別** | 検証ロジックの差異 |
| **対象** | `standardParser.ts:210-224` |
| **影響度** | **High** |
| **ステータス** | **有効** |

**発見内容:** 仕様では `(N)` 欠落時はエラーとするが、実装では出目の合算にフォールバック。改竄防止要件の欠陥。

- 影響は「あにまん掲示板で1dXダイスの(N)が省略されるケース」に限定
- **推奨:** 仕様を「NdX(N>=2)は(N)必須、1dXは(N)オプション」に更新
