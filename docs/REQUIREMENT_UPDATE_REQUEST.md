# Requirement Update Request (PM to Architect)

## Overview
ユーザーからの要望および実装フェーズでの課題発見に基づき、`REQUIREMENTS.md` の更新を依頼します。
本更新が完了次第、Engineer工程にて実装・修正作業を開始します。

## Requested Changes

### 1. Scene 1 Entry Logic (Mid-phase Reduction)
*   **Current Issue:**
    *   中盤ダイス回数を減少させた際（例: 2回→1回）、削除されるフェーズ（例: 中盤2）を「固有スキル発動位置」に指定していたエントリーのデータ不整合が発生する。
    *   現在はUIの制約（プルダウン項目の動的変化）により、意図せず「序盤」等に書き換わってしまうリスクがある。
*   **Requirement Update:**
    *   中盤回数の変更により、現在設定されている「固有発動位置」が存在しなくなる場合、その出走者の発動位置を **即座に `---` (未選択/null)** に強制リセットする仕様を明記してください。
    *   これにより「必須項目未入力」の状態を作り出し、エントリー確定をブロックして、GMに正しい再設定を促します（勝手なデフォルト値への変更を禁止）。

### 2. Scene 2 Confirm & Dice Output
*   **Current Issue:**
    *   エントリー確認リストの表示形式、およびダイス出力形式が古い要件のまま実装されている。
*   **Requirement Update:**
    *   **確認リスト:** 日本語表記（`逃げ / ギャンブル` 等）への統一。
    *   **ダイス出力:** 名前とダイス式の区切り文字を **全角スペース (U+3000)** (`　`) に統一。

### 3. Parser Specification (88-ch Support)
*   **Current Issue:**
    *   サブ会場として想定される別掲示板 (`88-ch`) のダイス形式に対応していない。
*   **Requirement Update:**
    *   `88-ch` 用のパーサー仕様を追加してください。
    *   **特徴:** 絵文字 `🎲` を含むフォーマット（例: `名前 🎲 dice1d100= result`）。
    *   **解析詳細:** 行頭の `🎲` や、名前と `🎲` の間のスペース有無など、表記揺れへの許容範囲も定義をお願いします。
